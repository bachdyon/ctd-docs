---
title: "Docker & môi trường"
description: "Chạy stack dev với Docker Compose và kiểm tra health."
---

## 1) Yêu cầu trước khi bắt đầu

- Cài **Docker Desktop** (hoặc Docker Engine + Compose plugin).
- Mở terminal tại thư mục gốc dự án.
- Đảm bảo các port mặc định chưa bị chiếm: `8080`, `15000`, `27017`, `5672`, `6379`, `6333` (tùy cách bạn chạy API/frontend — có thể chỉ cần một phần).

## 2) Cấu hình biến môi trường (`.env`)

Tạo file `.env` từ mẫu:

```bash
cp .env.example .env
```

Thiết lập tối thiểu nên kiểm tra:

```env
MONGO_URI=mongodb://mongo:27017
DATABASE_NAME=ctd
SECRET_KEY=your-secret
JWT_SECRET_KEY=your-jwt-secret
```

Nếu chạy API trên máy host cùng compose chỉ có Mongo/Redis/RabbitMQ/Qdrant, dùng `mongodb://localhost:27017` và các URL tương ứng trên `localhost`.

## 3) Chạy hệ thống bằng Docker Compose

```bash
docker compose up -d --build
```

Tùy `docker-compose.yml` của bạn, có thể chỉ khởi động hạ tầng; khi đó chạy API và frontend trên máy theo hướng dẫn riêng.

## 4) Kiểm tra trạng thái dịch vụ

```bash
docker compose ps
```

Kỳ vọng: các container định nghĩa trong file compose ở trạng thái `running` hoặc `healthy`.

## 5) Kiểm tra endpoint quan trọng

```bash
curl -i http://localhost:15000/health

# Frontend (nếu có)
curl -I http://localhost:8080
```

## 6) Xem log khi gặp lỗi

```bash
docker compose logs -f api
docker compose logs -f frontend
docker compose logs -f rabbitmq
```

(Bỏ các dòng không tồn tại service tương ứng trong compose của bạn.)

## 7) Dừng hệ thống

```bash
docker compose down
```

Xóa cả volumes (cẩn thận, mất dữ liệu local):

```bash
docker compose down -v
```
