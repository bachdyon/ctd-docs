## Mục tiêu

- Loại bỏ toàn bộ secret hard-code khỏi repo.
- Giảm rủi ro lộ thông tin cá nhân/tài chính trước khi public.
- Thiết lập quy trình để không tái diễn.

## Các nhóm dữ liệu cần kiểm tra

1. **API keys / tokens**: OpenAI, cloud storage, các provider tích hợp khác.
2. **Database URI chứa user/pass**.
3. **Mật khẩu mặc định, secret key JWT/Flask**.
4. **Thông tin cá nhân**: email, số điện thoại, endpoint nội bộ có thể truy vết.
5. **File test/lab/script mẫu** hay bị hard-code secret.

## Kế hoạch thực thi đề xuất

### Bước 1: Quét nhanh toàn repo

Dùng regex để tìm các pattern secret phổ biến, ví dụ:

```bash
rg -n "sk-|AKIA|AIza|mongodb\+srv://|client_secret|api_key|password\s*=|SECRET_KEY\s*="
```

### Bước 2: Phân loại kết quả

- **Critical**: secret thật còn dùng được (phải rotate ngay).
- **Medium**: dữ liệu nội bộ, thông tin nhận diện cá nhân.
- **Low**: chuỗi giả lập trong test.

### Bước 3: Xử lý theo mức độ

- Secret thật:
  - Gỡ khỏi code.
  - Chuyển sang env vars.
  - Rotate khóa ngay trên nhà cung cấp.
- URI chứa mật khẩu:
  - Đổi tài khoản DB mới.
  - Thu hồi user/password cũ.
- Password/secret mặc định:
  - Đặt giá trị trung tính (vd: `CHANGE_ME_IN_ENV`).

### Bước 4: Chặn tái diễn

- Thêm bước scan secret vào CI.
- Thêm `.env.example` và hướng dẫn rõ ràng.
- Quy ước: không commit `.env`, không hard-code key ở `labs/`.

## Những điểm đã xử lý trong repo này

- Bỏ OpenAI key hard-code trong `workers/videos/labs/try_whisper.py`.
- Bỏ MongoDB Atlas URI hard-code trong `api/labs/semantic_search.py`.
- Đã gỡ module thanh toán (PayOS / `api/routes/payment.py`) khỏi runtime.
- Bỏ API key built-in và root password mặc định cứng trong `settings.py`.

## Checklist trước khi public chính thức

- [ ] Rotate toàn bộ khóa cũ đã từng xuất hiện trong git history.
- [ ] Dọn lịch sử git nếu cần (BFG/git filter-repo).
- [ ] Bật bảo vệ nhánh + bắt buộc review PR.
- [ ] Bật secret scanning trên GitHub/GitLab.
