## Bước 1: Chọn đúng file cài đặt

- **Mac M1, M2, M3, M4 (Apple Silicon):** tải file **Contenttudong_0.1.0_aarch64.dmg**
- **Mac Intel:** tải file **Contenttudong_0.1.0_x64.dmg**

Nếu cài nhầm bản (ví dụ dùng bản Intel trên Mac M1), app sẽ báo "Ứng dụng không được hỗ trợ trên máy Mac này" — khi đó hãy tải đúng bản theo loại máy ở trên.

---

## Bước 2: Mở file DMG

Double-click vào file `.dmg` vừa tải. Cửa sổ cài đặt sẽ mở ra với biểu tượng **Contenttudong** và thư mục **Applications**.

---

## Bước 3: Kéo app vào Applications

Kéo biểu tượng **Contenttudong** vào thư mục **Applications** (hoặc kéo vào biểu tượng Applications trong cùng cửa sổ).

Chờ copy xong. Sau đó có thể đóng cửa sổ DMG và đẩy ổ đĩa ảo (Eject) ra.

---

## Bước 4: Mở app lần đầu

1. Mở **Finder** → **Applications** → tìm **ContentTuDong** (hoặc **Contenttudong**).
2. **Chuột phải** vào app → chọn **Mở** (Open).
   - Lần đầu macOS có thể hiện cảnh báo "nhà phát triển không xác định". Chọn **Mở** để tiếp tục.
   - Nếu chỉ double-click, macOS có thể chặn; dùng chuột phải → Mở là được.
3. Lần đầu chạy, app sẽ tự tải thêm một số công cụ (MongoDB, Node, FFmpeg, Chrome…) — có thanh tiến trình, đợi đến khi xong.
4. Khi màn hình chính hiện ra, cài đặt hoàn tất.

---

## Lưu ý

- **Dung lượng:** sau khi cài, app và dữ liệu nằm trong thư mục **~/ContentTuDong/** (trong thư mục người dùng). Cần vài trăm MB cho lần chạy đầu.
- **Cập nhật:** tải bản DMG mới, cài đè (kéo vào Applications và chọn Replace). Nên thoát app trước khi cài đè.
- **Gỡ cài đặt:** xóa app trong Applications và xóa thư mục dữ liệu **~/ContentTuDong/** (trong Finder: Go → Home, mở thư mục **ContentTuDong** và xóa; hoặc trong Terminal chạy: `rm -rf ~/ContentTuDong`).

## Debug khi gặp lỗi (dành cho support)

Mở **Terminal** (Spotlight gõ "Terminal"), chạy lần lượt và gửi kết quả cho người hỗ trợ.

**1. Xem log app (50 dòng cuối):**
```bash
tail -50 ~/Library/Logs/com.contenttudong.app/ContentTuDong.log
```

**1b. Xem log realtime (toàn bộ log rồi theo dõi dòng mới; thoát: Ctrl+C):**
```bash
tail -n +1 -f ~/Library/Logs/com.contenttudong.app/ContentTuDong.log
```

**2. Kiểm tra API có chạy không (kỳ vọng thấy "200 OK"):**
```bash
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:15000/health
```

**3. Xem port 15000 và 27018 có process nào đang dùng:**
```bash
lsof -i :15000 -i :27018 2>/dev/null || echo "Không có process nào trên 15000/27018"
```

**4. Xem thư mục dữ liệu app (có tồn tại, có file gì):**
```bash
ls -la ~/ContentTuDong 2>/dev/null || echo "Thư mục chưa tồn tại"
```

**5. Nếu app treo/không mở được — tắt hết process rồi mở lại app:**
```bash
pkill -f ContentTuDong; pkill -f contenttudong; pkill -f mongod
```
Sau đó mở lại ContentTuDong từ Applications.

**6. Gửi file log cho người hỗ trợ**

Chạy lệnh sau để copy file log ra **Desktop** (tên file: `ContentTuDong-log.txt`):

```bash
cp ~/Library/Logs/com.contenttudong.app/ContentTuDong.log ~/Desktop/ContentTuDong-log.txt
```

Sau đó trên Desktop sẽ có file **ContentTuDong-log.txt**. Người dùng có thể:
- Đính kèm file này vào email, hoặc
- Kéo thả vào Zalo/Telegram/Google Drive rồi gửi link, hoặc
- Upload lên Google Drive / WeTransfer rồi gửi link cho bạn.
